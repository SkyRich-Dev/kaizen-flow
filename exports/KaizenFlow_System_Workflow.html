KaizenFlow - System Workflow Document

Version: 1.0 | Date: December 2025

1. Workflow Overview

The KaizenFlow system implements a comprehensive 7-stage approval workflow for processing Kaizen improvement requests. The workflow ensures proper evaluation by all relevant departments and escalates to senior management based on cost thresholds.

Complete Workflow Flow

DRAFT → OWN MANAGER → OWN HOD → CROSS-MANAGERS → CROSS-HODs → AGM (if applicable) → GM (if applicable) → APPROVED/REJECTED

2. Detailed Workflow Stages

Stage 1: DRAFT

Status: DRAFT

Actor: Initiator (any department employee)

Actions:

  • Create new kaizen request
  • Fill in all required fields (title, station, issue description, cost estimate, etc.)
  • Attach supporting documents (PFMEA, check sheets, photos)
  • Save as draft or submit for approval

Next Stage: PENDING_OWN_MANAGER (on submit)

↓

Stage 2: OWN DEPARTMENT MANAGER REVIEW

Status: PENDING_OWN_MANAGER

Actor: Manager of the initiator's department

Actions:

  • Review request details and feasibility
  • Verify cost estimates
  • Add comments/feedback
  • Approve or Reject with reason
Decision Points:
  • Approve: Moves to PENDING_OWN_HOD
  • Reject: Moves to REJECTED status with reason
↓

Stage 3: OWN DEPARTMENT HOD APPROVAL

Status: PENDING_OWN_HOD

Actor: Head of Department (HOD) of the initiator's department

Actions:

  • Final department-level review
  • Verify manager's assessment
  • Confirm budget availability
  • Approve or Reject
Decision Points:
  • Approve: Moves to PENDING_CROSS_MANAGER
  • Reject: Moves to REJECTED status
↓

Stage 4: CROSS-DEPARTMENT MANAGER EVALUATION

Status: PENDING_CROSS_MANAGER

Actors: Managers from ALL OTHER departments (4 managers)

Purpose: Risk assessment from each department's perspective

Actions per Manager:

  • Answer department-specific evaluation questions
  • Rate risk level (Low/Medium/High) for each question
  • Provide mandatory remarks for "No" answers or high-risk items
  • Accept or Reject the request

Evaluation Questions by Department:

DepartmentQuestionsFocus Areas
Maintenance7Equipment maintenance, safety guards, spare parts, training
Production5Capacity impact, compatibility, downtime, targets
Assembly6Process flow, worker safety, instructions, cycle time
Admin4Policy compliance, approvals, documentation
Accounts5Budget approval, vendor payments, cost tracking, ROI
Completion Criteria:
  • All 4 cross-department managers must complete their evaluation
  • Any rejection by a manager moves request to REJECTED
  • All acceptances move to PENDING_CROSS_HOD
↓

Stage 5: CROSS-DEPARTMENT HOD EVALUATION

Status: PENDING_CROSS_HOD

Actors: HODs from ALL OTHER departments (4 HODs)

Purpose: Senior-level risk validation from each department

Actions per HOD:

  • Review manager's evaluation
  • Answer department-specific evaluation questions
  • Rate risk levels and provide remarks
  • Accept or Reject
Completion Criteria:
  • All 4 cross-department HODs must complete their evaluation
  • Any rejection moves request to REJECTED
  • All acceptances trigger cost-based routing
↓

Stage 6: AGM APPROVAL (Conditional)

Status: PENDING_AGM

Actor: Assistant General Manager

Trigger Conditions (any one):

  • Cost estimate exceeds ₹50,000
  • Request requires process addition
  • Request requires manpower addition

Actions:

  • Review all previous evaluations
  • Assess organizational impact
  • Verify resource availability
  • Approve or Reject
Decision Points:
  • Approve + Cost ≤ ₹100k: Moves to APPROVED
  • Approve + Cost > ₹100k: Moves to PENDING_GM
  • Reject: Moves to REJECTED
↓

Stage 7: GM APPROVAL (Conditional)

Status: PENDING_GM

Actor: General Manager

Trigger Condition:

  • Cost estimate exceeds ₹100,000

Actions:

  • Final executive review
  • Strategic alignment assessment
  • Budget approval
  • Approve or Reject
Decision Points:
  • Approve: Moves to APPROVED
  • Reject: Moves to REJECTED
↓

Stage 8: FINAL STATUS

Status: APPROVED or REJECTED

Actions:

  • Notification sent to initiator
  • Audit log updated
  • Request archived for reporting

3. Cost-Based Routing Logic

Cost RangeApproval PathFinal Approver
₹0 - ₹50,000Own Manager → Own HOD → Cross-Managers → Cross-HODsCross-Department HODs
₹50,001 - ₹100,000...Cross-HODs → AGMAGM
Above ₹100,000...AGM → GMGM
Any cost + Process/Manpower Addition...Cross-HODs → AGM (→ GM if >₹100k)AGM or GM

4. Rejection Handling

A request can be rejected at any stage. When rejected:

  • The rejecting user must provide a reason
  • Request status changes to REJECTED
  • Initiator is notified with the rejection reason
  • The rejecting department is recorded
  • Full audit trail is maintained
